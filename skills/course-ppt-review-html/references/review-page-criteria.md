# Review Page Criteria

## Coverage

Use slide order to reconstruct the lecture logic before writing. A good review page should answer:

- What is the module about?
- What are the major methods, signals, mechanisms, equations, or models?
- Which concepts are needed for compare/contrast questions?
- Which terms are low-frequency but useful for understanding figures or equations?
- Which slide images make abstract content easier to remember?

Do not cap terminology or short answers unless the user explicitly asks for a fixed exam drill.

## Sections

Recommended structure:

1. Header with module name, source PDF/PPT, and coverage status.
2. Logic map by slide ranges.
3. Visual explanation section with selected slide screenshots.
4. Terms grouped by slide range or concept group.
5. English short answers with Chinese sentence translations.
6. Method comparison cards.
7. Misconception or "do not confuse" cards.
8. Memorization sequence for two or three review passes.

## Image Selection

Select images that explain something hard to memorize:

- method comparison figures,
- system/block diagrams,
- signal traces,
- anatomical or device diagrams,
- tables that map concepts,
- equations or model diagrams,
- parameter or mechanism timelines.

Skip pages that are only title, author biography, bibliography, or decorative context unless the user asks for completeness.

## Image Explanation

For each important slide image, write more than a caption:

- what the slide is trying to teach,
- how to read the figure or table,
- what the learner must remember,
- one exam-ready English sentence,
- one common misconception to avoid.

The goal is that a learner who cannot understand the PPT page alone can still follow the HTML explanation.

## English Answer Quality

Each short answer should be:

- grammatical and complete,
- directly useful in an exam,
- short enough to memorize,
- grounded in slide content,
- translated into Chinese as a full sentence.

Prefer answer stems such as:

- "Compared with X, Y is more/less ..."
- "This method measures ..."
- "At the membrane level, ..."
- "The model explains ... by ..."
- "This matters because ..."

## Validation

Before delivery, check:

- image files exist,
- image paths load through the same URL/file context as the HTML,
- every English short answer has a Chinese translation,
- every slide-grounded answer has source slide labels,
- no placeholder text remains,
- no body-level horizontal overflow at mobile width,
- local browser console has no errors.
